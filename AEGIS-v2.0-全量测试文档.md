## 1. 测试概述

### 1.1 文档目的

本文档定义 AEGIS v2.0 多模型协作架构的全量测试范围、测试策略、测试用例和通过标准，确保重构后的系统满足功能需求、性能指标和质量标准。

### 1.2 被测系统简介

AEGIS v2.0 是重构后的多模型协作编码助手，核心特性包括：

- **多模型动态路由:** 根据任务类型自动选择最优模型组合
- **协作模式引擎:** 支持 Pair/Swarm/Pipeline/MoA 四种协作模式
- **智能任务分类:** 自动识别任务类型（架构/编码/审查/调试）
- **成本控制:** 实时追踪多模型调用成本
- **会话管理:** 多模型上下文共享和状态同步

### 1.3 测试目标

| 目标编号 | 目标描述 | 成功标准 |
|---------|---------|---------|
| T-001 | 验证多模型路由准确性 | 任务分类准确率 ≥ 95% |
| T-002 | 验证协作模式正确执行 | 四种模式执行成功率 100% |
| T-003 | 验证模型间上下文共享 | 上下文同步延迟 < 100ms |
| T-004 | 验证成本控制功能 | 成本追踪误差 < 5% |
| T-005 | 验证系统性能 | 简单任务响应 < 30s，复杂任务 < 5min |
| T-006 | 验证异常恢复能力 | 单模型故障不影响整体任务完成率 ≥ 99% |

## 3. 测试策略

### 3.1 测试金字塔

```
                    ┌─────────┐
                    │  E2E    │  ← 端到端测试 (10%)
                    │  Tests  │     完整用户场景
                    ├─────────┤
                    │Integration│ ← 集成测试 (30%)
                    │  Tests  │     模块间协作
                    ├─────────┤
                    │  Unit   │ ← 单元测试 (60%)
                    │  Tests  │     单个函数/类
                    └─────────┘
```

### 3.2 测试方法

| 测试类型 | 方法 | 工具 | 覆盖率目标 |
|---------|------|------|-----------|
| 单元测试 | 白盒测试 | pytest | ≥ 80% |
| 集成测试 | 灰盒测试 | pytest + 模拟器 | 核心流程 100% |
| 性能测试 | 负载测试 | locust + custom | 基准对比 |
| 兼容性测试 | 黑盒测试 | 多环境矩阵 | 主要配置 |
| 异常测试 | 故障注入 | chaos testing | 关键路径 |

### 3.3 测试数据策略

| 数据类型 | 来源 | 管理方式 |
|---------|------|---------|
| 模型响应 | Mock Server | 录制真实响应，支持回放 |
| 任务样本 | 历史数据 | 100+ 真实任务分类标注 |
| 成本数据 | 模拟 + 真实 | 固定汇率和定价模型 |
| 配置文件 | 版本控制 | 多环境配置模板 |
