# PRD Template (AEGIS Standard)

## 1. Problem Statement
What user pain point or business opportunity does this address?

## 2. User Stories
Format: As a [persona], I want [action], so that [outcome].

- US-1: ...
- US-2: ...

## 3. Acceptance Criteria
Format: Given [context], When [action], Then [expected result].

- AC-1: ...
- AC-2: ...

## 4. Non-Functional Requirements
- Performance: ...
- Security: ...
- Scalability: ...
- Compliance: ...

## 5. Out of Scope
Explicitly list what will NOT be built in this iteration.

## 6. Open Questions
Questions that must be resolved before architecture is finalized.
